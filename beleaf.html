BeLeaf Case Study

A native app for iOS & Android to
keep you on top of your plant care

UX/UI Case Study

Project Overview

Project Goals

To design a native iOS and Android app that helps people keep on top of watering, repotting, and fertilizing their house plants by sending them push notification reminders.

Context

There are a lot of competitors on the field, but they tend to have an overload features and hide the good stuff behind a pay wall. This app aims to have a powerful basic feature set that doesn’t overwhelm and is simple to use.

Users

From fledgling plant parents to veteran gardeners, this app will be useful to anyone who owns a plant.

MVP Key Features

  • Ability to enter plants that the user owns
  • Provide information on watering level, soil type, fertilization schedule, and light intensity
  • Set notifications for watering, fertilizing, and repotting
  • Ability to search for plants

Information Architecture

Three user flows were necessary to build the MVP:

“I want to set up a new plant.”

“I want to be able to see reminders and care tips for my plant.”

“I want to view all my upcoming reminders for plant care.”

Using the latest human interface guidelines from Apple and Material Designs, I created two versions of the app that use standard UI elements from each mobile operating system. This ensures the app will feel familiar to the user even before they begin using it. Once I had applied the OS standards to the app, I was able to apply visual stylings that are specific to the app.

Icons

The visual style of the app draws inspiration from a cherry blossom, using organic curves, soft drop shadows, and pale pink & greens for the color scheme. The color scheme is soothing and complimentary to the subject matter without drawing attention away from the plants.

User Testing

I tested my high fidelity wireframes on 5 users by taking them through the 3 userflows of the MVP. Everyone expressed their need for an app of this type, and praised the visual design. They also provided valuable feedback on some pain points in the app.

Retrospective

What went well

I’m getting much faster at producing wireframes. I think the color scheme came out quite lovely. It was an interesting exercise to create an app that strictly adhered to iOS/Material Design guidelines before branching off into a different visual direction that tied the two together. My users were all really happy with the app and said it’s something they would like to use.

Challenges overcome

I went through a couple design iterations with the date picker before deciding on one that I liked, and I had a hard time finding examples of them to emulate in Material Design’s specific style. I overused gradients for visual appeal, without considering that they implied clickability; user feedback was crucially important to pointing this out to me.

What can be improved

I wish that I had been less destructive while editing the UI elements that I took from iOS and Material Design resources, in the future I’ll use the visibility toggle on elements rather than deleting. I had a really hard time deciding between using Sketch’s built in prototyping tools and InVision, because each of them were missing features that I wanted in my prototypes. Next time I want to explore using Figma for prototyping.
